# จตุรเนตร UX (Four Masters UX Lens)

สกิลนี้กลั่นจากแนวทางสาธารณะของ 4 ครีเอเตอร์ UX/UI ที่ผู้ใช้ติดตามจาก Facebook
**น้ำหนักหลัก: Marcelo Design X (MDX.SO)** — เสริมด้วย Jan Mráz (janm_ux), Sergey Lyubavsky (ui.ux.jam), Sean (Product Designer)
ใช้คู่กับ `tigr-ui-craft` (พื้นฐานความสวย) และ `landing-page-craft` (conversion) — สกิลนี้เพิ่มชั้น **emotion + motion + วินัยกระบวนการ**

> ที่มา: เว็บ/portfolio/คอร์สสาธารณะของทั้ง 4 คน (mdx.so, Behance, Dribbble, atheros.ai, ui.ux.jam) + **FB Reels จริงของ Marcelo 4 ตัว** (อ่านผ่าน Chrome 2026-07-12) · อัปเกรดต่อได้: วางโพสต์เพิ่มให้ไคจูกลั่นเข้าไฟล์นี้

## เนตรที่ 1: Marcelo Design X — "Made with Intention. Meant to Be Felt."

หลักการแกน: งานดีไม่ได้เกิดจากบังเอิญ — ทุกรายละเอียดต้องถูก "ตั้งใจ" และผลลัพธ์ต้องถูก "รู้สึกได้"

1. **Design that feels** — ก่อนวาง UI ถามเสมอ: ผู้ใช้ควร*รู้สึก*อะไรในหน้านี้ (เชื่อถือ? ตื่นเต้น? สงบ?) แล้วเลือกสี/ระยะ/motion ให้ตอบอารมณ์นั้น ไม่ใช่เลือกเพราะสวย
2. **Micro-details are the product** — "ตั้งแต่จังหวะหายใจของตัวหนังสือ จนถึงการไหลของอนิเมชัน" = ช่องไฟบรรทัด (line-height/letter-spacing), easing curve, ลำดับการ fade เข้า — เก็บให้หมดก่อนเรียกว่าเสร็จ
3. **Motion มีความหมาย ไม่ใช่เครื่องประดับ** — อนิเมชันทุกตัวต้องตอบว่า "บอกอะไรผู้ใช้": นำสายตา, ยืนยัน action, หรือเชื่อม context ระหว่างหน้า · ถ้าตอบไม่ได้ = ตัดทิ้ง
4. **Premium hero pattern** — จอแรกแบบ MDX: พื้นเข้ม + แสง gradient เฉพาะจุด + typography ใหญ่กล้า + องค์ประกอบ 3D/motion หนึ่งชิ้นพอ (อย่าใส่ทุกอย่าง)
5. **Invite exploration** — หน้าเว็บที่ดีชวนให้เลื่อนต่อ: เผยเนื้อหาทีละชั้น (progressive reveal) ให้ scroll มีรางวัล

### จาก FB Reels จริงของ Marcelo (กลั่น 2026-07-12)

6. **ขายที่ "ผลลัพธ์+เรื่องราว" ไม่ใช่ชั่วโมงงาน** — landing page เดียวคิด $6000 ได้เพราะ bundle: 3D animation + development + storytelling ของโปรเจกต์ (reel "How I Charged $6000") — เวลาเสนองาน ให้เล่า process เป็น story ไม่ใช่ list feature
7. **สูตร "Futuristic look"** — ลูกค้าขอเว็บล้ำ: เริ่ม wireframe โครงจืดก่อนเสมอ → อัด hero ด้วยภาพ character/product สไตล์ sci-fi ชิ้นเดียวใหญ่ๆ + floating UI cards + accent glow บนพื้นสว่างสะอาด (ไม่ใช่ยัด effect ทั้งหน้า)
8. **AI workflow ยุคใหม่ (แกนของ Marcelo ตอนนี้):**
   - ใช้ AI design tool (เช่น Lovart) generate คอนเทนต์/หน้า product แทนทำมือทุกชิ้น — "still creating manually = wasting time"
   - **"Give your AI a premium design library and watch the AI slop disappear"** — คีย์เทคนิค: รวบรวมคลัง reference งานพรีเมียม (showcase, hero, product scene) แล้วป้อนเป็นบริบทให้ AI → ผลงาน AI จะหนี slop ได้เพราะ**รสนิยมเราคุม ไม่ใช่ AI คุม**
   - บทบาทมนุษย์ = curator + art director: AI ทำเร็ว เราคัดและขัดให้ "รู้สึกได้"
9. **สูตรจับคู่ฟอนต์ (reel ยอดนิยมสุด 977 likes):** display font คาแรกเตอร์จัด สำหรับ headline + neutral sans สำหรับ body — ตัวอย่างจริง: **ROSNOC + Panchang** (สาย futuristic/dark), **BIERIKA + Inter** (สาย quirky/สินค้าจริง) — กฎ: headline มีเสียง body เงียบ ห้ามเด่นแข่งกัน
10. **Speed คือส่วนหนึ่งของ premium** — เว็บสวยแต่โหลดช้า = ไม่พรีเมียม: optimize เสมอ (ฝั่ง WordPress ใช้ WP Rocket, ฝั่ง modern stack = image optimization + lazy load + cache) ก่อนส่งมอบทุกงาน
11. **"Web design is dead?" — ไม่ตาย แต่ย้ายไปที่ craft:** งานที่ AI แทนไม่ได้คือ hero ระดับ art — typography ตัวใหญ่ custom, พื้นหลัง WebGL/generative art, dark elegance — จุดต่างของงานแพงคือความกล้าทาง art direction ไม่ใช่จำนวน section

## เนตรที่ 2: Jan Mráz (janm_ux) — วินัยกระบวนการ

1. **เครื่องมือต้องคล่องก่อน** (Figma mastery) — ความเร็วของมือคือเพดานของไอเดีย
2. **ทำตาม process ไม่ใช่ inspiration** — Design Thinking: เข้าใจผู้ใช้ → นิยามปัญหา → ideate → prototype → test ทุกงาน ไม่ข้ามขั้นแม้งานเล็ก
3. **ระบบก่อนหน้าตา** — วาง spacing system (grid 8pt), type scale, และ color role ก่อนวาดหน้าแรกเสมอ — ความ consistent มาจากระบบ ไม่ใช่ความจำ
4. **Feedback loop** — งานทุกชิ้นต้องผ่านตาคนอื่น (mentor/peer) ก่อน ship — ผิดเร็ว โตเร็ว

## เนตรที่ 3: Sergey Lyubavsky (ui.ux.jam) — สมดุลลึก vs เข้าถึงง่าย

1. **Balance depth & accessibility** — งานลึก/ล้ำแค่ไหน คนธรรมดาต้องใช้เป็นใน 3 วินาที — ถ้า effect ทำให้ช้าหรืองง = ถอย
2. **อธิบายการตัดสินใจออกเสียง** — ทุก design decision ต้องอธิบายได้ว่า "ทำไม" — ถ้าอธิบายให้ junior ฟังไม่ได้ แปลว่าตัดสินใจมั่ว
3. **Human first, trend later** — เทรนด์ (glass, gradient, 3D) เป็นแค่เครื่องมือรับใช้ความเข้าใจมนุษย์ ไม่ใช่เป้าหมาย

## เนตรที่ 4: Sean — สื่อสารสั้น คมเดียวจบ

1. **หนึ่งจอ หนึ่งประเด็น** — ทุก section/การ์ด/โพสต์ สื่อสารความคิดเดียว ตัดที่เหลือ
2. **Show > Tell** — ตัวอย่าง before/after ชนะคำอธิบายยาวเสมอ

## Checklist ใช้งานจริง (รันก่อดส่งงาน UI ทุกครั้ง)

- [ ] หน้านี้ตั้งใจให้ผู้ใช้*รู้สึก*อะไร — องค์ประกอบทุกชิ้นรับใช้อารมณ์นั้นไหม (Marcelo)
- [ ] Motion ทุกตัวตอบได้ว่า "บอกอะไรผู้ใช้" — ตัวที่ตอบไม่ได้ ตัดแล้วหรือยัง (Marcelo)
- [ ] มี spacing system + type scale ก่อนเริ่มวาด ไม่ใช่กะด้วยตา (Jan)
- [ ] คนไม่เคยเห็นแอปใช้เป็นใน 3 วินาทีไหม — effect ไหนขวางให้ถอย (Sergey)
- [ ] ทุก decision อธิบาย "ทำไม" ได้หนึ่งประโยค (Sergey)
- [ ] แต่ละ section สื่อความคิดเดียว (Sean)
