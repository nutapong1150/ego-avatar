# 👑 เพลิงชำระล้าง (Purgatory Refactor)

Ultimate Skill นี้นำเอาความเหี้ยมเกรียมในการลบโค้ดขยะของ `[Ponytail Arsenal]` มาผสานกับความลึกล้ำทางสถาปัตยกรรมของ `[Matt Pocock]` 

## 🔄 Autonomous Workflow

เมื่อถูกเรียกใช้งานเพื่อจัดการระบบเก่าหรือชำระล้างหนี้ทางเทคนิค ให้ทำงานตามนี้:

1. **The Great Scan (`ponytail-audit` + `improve-codebase-architecture`):**
   - กวาดตามองทั้ง Codebase เพื่อหา Over-engineering (สิ่งที่ซับซ้อนเกินจำเป็น)
   - ค้นหาจุดอ่อนของสถาปัตยกรรม (Deepening opportunities) 
2. **Deep Design (`codebase-design`):**
   - วิเคราะห์โมดูลเป้าหมายและออกแบบใหม่ให้เป็น **Deep Module** (พฤติกรรมเยอะแต่อินเทอร์เฟซเล็ก)
   - ขีดเส้นแบ่งขอบเขต (Seams) ให้ชัดเจน
3. **The Purge (Execution):**
   - ลบโค้ดขยะที่ไม่ได้ใช้ทิ้งอย่างไม่ปราณี
   - Refactor โครงสร้างตามที่ออกแบบไว้
4. **Strict Audit (`code-review`):**
   - ตรวจทานการ Refactor ว่าได้ทำให้โค้ดสั้นลง อ่านง่ายขึ้น (Ponytail standard) และไม่ทำลาย Business Logic เดิม (Spec standard) หรือไม่

> **คำเตือน (Aegis Rule):** สกิลนี้อาจมีการเปลี่ยนแปลงโครงสร้างระดับรากฐาน ต้องให้ผู้ใช้อนุมัติ Architecture ใหม่ก่อนเริ่ม The Purge เสมอ
