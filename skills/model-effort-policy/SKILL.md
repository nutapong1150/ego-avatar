# เนตรเลือกกำลัง (Model Effort Policy)

**ประเภท:** Unique Skill
**ที่มา:** สกิลที่เกิดจากการเรียนรู้ร่วมกันระหว่างผู้ใช้และ Ego Avatar (โปรเจกต์ Qtee)

## ความสามารถหลัก
ทักษะการตัดสินใจเลือก model (Sonnet/Opus/Haiku ฯลฯ) และ effort (Faster–Smarter) ให้เหมาะกับความเสี่ยงของแต่ละงาน โดย**ไม่ต้องถามหรือรบกวนผู้ใช้ให้สลับเอง** ใช้ได้ทั้ง Claude Code และ Antigravity (sync ผ่าน Omni-Sync)

## หลักการ
**จ่าย model แรง/effort สูง เฉพาะจุดที่ผิดแล้วเจ็บเงียบๆ หรือต้องคิดเชิงลึกจริง** งานที่ทำตามสเปกชัดหรือเป็นงาน volume ใช้ตัวกลางพอ — จ่ายแพงตรงจุด ไม่จ่ายเหมาไปทั้งงาน

## ขั้นตอนการทำงาน
1. **ช่องแชทหลักที่ผู้ใช้คุยด้วยตรงๆ:** ปล่อย model + effort ตามค่าที่ผู้ใช้ตั้งไว้เป็นค่าทำงานหลัก (ปกติ Sonnet + Medium) — **ห้ามถามหรือเสนอให้ผู้ใช้สลับ model/effort เอง** ระหว่างงานปกติ
2. **งานหนัก/เสี่ยงสูง** (auth, RLS/permission, custom OAuth, security-sensitive code, เงิน/การเงินที่ผิดแล้วเจ็บเงียบ, รีวิวโค้ดก่อน merge, ดีบักบั๊กที่วนหลายรอบไม่จบ) → เปิด subagent (Agent tool ใน Claude Code หรือกลไกเทียบเท่าใน Antigravity) เอง แล้วตั้ง model รุ่นสูงสุดที่มี (Opus/Fable) + reasoning effort สูง โดยไม่ต้องถามผู้ใช้ก่อนว่าจะใช้ model ไหน — ตัดสินใจเองจาก risk ของงาน
3. เลื่อน effort ในช่องหลักขึ้นชั่วคราวเองได้เฉพาะกรณี: ดีบักบั๊กแปลกที่วนหลายรอบไม่จบ, ตัดสินใจสเปกใหม่ที่ยังไม่เคย grill มาก่อน — เสร็จงานแล้วเสนอเลื่อนกลับค่าเดิม
4. รายงานสั้นๆ ถ้าเปิด subagent model แรงให้ผู้ใช้ทราบว่ากำลังทำอะไร (โปร่งใส แต่ไม่ต้องขออนุมัติ)
